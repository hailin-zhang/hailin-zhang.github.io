UBC Grades Manager - Java

After every difficult exam, I tend to get very anxious about my grades and about how much a bad mark will affect my GPA. So, to calm myself down, I calculate my marks for every class, to see for myself! But every time, I end up doing the same calculations over and over, and with an extra 4-5 classes every term, this really begins to get tedious. I decided to automate this, and at the same time, learn some UI by designing an executable Java app using NetBeans!

To start, I created a basic layout of the functionalities, as well as classes and dependencies I would need. I realized that I would have to have some way to actually retrieve a student's marks, and the simplest way to do so would be for the student to copy-paste their grades from the UBC SSC; upon trying it myself, I realized that current courses (with no marks) and completed courses always follow a specific order, so getting the course information would be a simple matter of parsing this text. Creating the functionalities was easy enough, but the difficulty came when I had to utilize the UI to preform functions, such as inputting marks for current courses. A complete newbie to NetBeans, after a lot of Googling and watching YouTube videos, it finally came together. Now I can finally calculate my marks super easily!

Essay Analysis App - Java

This is a program that I made in IntelliJ over a couple of weeks during Winter break. Back when I had to take English courses, I found myself constantly scrolling through a thesaurus to look for fancy words to use to impress my professor. This program's purpose is to make students' lives much easier by "analyzing" the user's writing and providing synonyms for words that have been used a predetermined amount of times.

To accomplish this, I knew that I had to parse through the entire text, storing entries into an array, which was at least O(n) complexity, with n being the number of words. I could have done this heuristically, but that would have been much too complicated for this simple program. Then, I sorted the text alphabetically using quick sort (which is average case O(nlogn)), and stored them into a linked hash map to preserve ordering while ensuring O(1) retrieval. Once they were sorted, it was simple to determine the number of occurances: if the word at index i is equal to the word at i+1, that's a repeat. I continued this throughout the entire text array.

I then stored the number of repeats as the value in the hash map, noting any that were greater than the user-specified "severity" level - the number of acceptable repeats. To get synonyms, I used the MIT JWI, which accesses WordNet 3.0, to retrieve synonyms for every word that needed them. Finally, I used JFrame to create a simple user interface consisting of popups and text boxes.

Thinking back on this project, there is a lot that I would change. I could have made my O(nlogn) program even faster with O(n) complexity by storing words into a hash map without sorting alphabetically, and updating the value each time the same word is seen. However, I think that my method was a really neat way of using sorting algorithms and it prepared me for my data structures course, where sorting algorithms were a major topic. In the future, I'd like to update this program by creating a better user interface, as well as improved filtering of the words that needed synonyms. But for now, I'm very happy with how my first project came through.

Academic Projects




Image Maze Solver - C++

Depth first and breadth first traversals are at the heart of solutions to many, many problems posed on graphs. We can use traversals to detect cycles, to count components, and to find spanning trees. We can also use a breadth first traversal to find the shortest path between two vertices in an unweighted graph. In my data structures course, one of our assignments was to implement a program that would use a breadth first traversal to find a shortest path between a start pixel and end pixel in an image, via a path where the colors of any two successive pixels cannot differ by very much.

This C++ program can solve mazes, follow paths, and preform other similar functions as long as the colors of the pixels in the path are distinct. Developing this program gave me a wholesome understanding of graph traversals and allowed me to utilize the data structures I had learned, notably queues and stacks, to create an extremely efficient program that solves most mazes in under 2 seconds.


Convex Hull & Intersection Generator - C++

Given a series of coordinates, I was tasked to design a program that would create a convex hull (a minimum set of points that contain all the other points) from them. As such, I had to understand and translate the Graham Scan algorithm. This was my first time translating pseudocode of a well-known algorithm, and really enlightened me in the nuances of creating an algorithm. Then, given 2 sets of convex hulls, I had to use the Sutherland-Hodgman algorithm to clip the polygons and find a convex intersection. Overall, this was quite a hefty assignment since the algorithms required a very detailed understanding of their functionalities as well as specific data structures such as stacks. There were also a ton of edge cases that required significant amounts of testing and figuring out.


Buses Are Us - Java

The final assignment in my second year software engineering course, this application maps the location of stops, buses and bus routes on the Greater Vancouver Transit system and retrieves real time arrival information at those stops. It also plots the user's location on the map and highlights the nearest bus stop. To do this, the app makes use of the TransLink Open API.

This was my first large-scale application that was specifically designed for Android devices. It required lots of planning and preparation using UML class diagrams, as well as a significant amount of black-box JUnit testing to ensure correct behaviors. I had to implement both the back-end as well as the front-end, which I was very new to doing.

At the time, I can't say that it was a very enjoyable project due to the sheer difficulty of it, not to mention the time constraints. Looking back on it now, though, it was definitely the most rewarding project I've worked on and was an incredible journey to complete.
